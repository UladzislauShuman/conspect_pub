Что Плохо
- Field Injection в Spring
- Создавать бины через new (иначе не будут Proxy)
- Изменяемое состояние в Singleton
- Double для денег
- Огромные @Transactional
- N+1
- Фильтрация в памяти (выгрузить все Из БД И потом фильтровать)
- catch общий Exception 
- Optional.get() или !!
    - в принципе -- не забываем использовать Optional
- явное приведение типов
- System.out.println вместо логгера
- в принципе -- мало логгирования. ну или слишком много
- Магические строки 
- вложенные IF
- Отсутствие инкапсуляции
- REST и названия методов (/getProducts)
- Repository и Serivce одновренменно используются в Controller
- бизнес логика в Controller
- можно было бы выделить несколько dto-классов (API, Бизнес, Модель БД). для каждой отдельный маппер
- желательно не забывать про слово final 
- правильно использовать exception-ы (названия, не обобщенное кидать)
- убрать лишние аннотации
- статусы и прочее наверное стоит через enum или sealed классы делать
- id наверное стоит передавать как pathVariable, а не как RequestParam
- важен контекст
    - например: если микросервисы, то -- "а что будет, если упадет сеть?". типо -- как обработать retry ("чтобы при дублировании одинаковых запросов не снялись дважды деньги")
- getAll, FindAll
    - нельзя брать ВСЕ за раз -- нужно как-то фильтровать или pageable делать
- service не должен заниматься mapping-ом. в принципе -- разделение отвественности
- "к этим данным очень часто делаеют запрос. что бы вы сделали"

## Какие новые классы для работы с датами появились в java 8?

- `LocalDate` -- дата без времени и временных зон 
- `LocalTime` -- время без даты и временных зон
- `LocalDateTime` -- дата и время без временных зон, в системе ISO-8601. объединяет LocalDate и LocalTime. Время хранится с точностью до наносекунд. есть множество удобных методов, таких как plusMinutes, plusHours, isAfter, toSecondOfDay и т.д.
- `ZonedDateTime` -- дата и время с временной зоной. аналог java.util.Calendar. Имеет самый полный объем информации о временном контексте из всех классов. как нетрудно видеть, вклюачет в себя временную зону, и тем самым все оперцаии с временными сдвигами он проводит с учетом временной зоны
- `DateTimeFormatter` -- форматирует даты в строки и наоборот, только для классов java.time
- `Instant` -- колличество секунд с Unix epoch time (полночь 1 января 1970 UTC)
- `Duration` -- продолжительность в секундах и наносекундах 
- `Period` -- период времени в годах, месяцах, днях
- `TemporalAdjuster` -- корректировщик дат (может получить дату следующего понедельника и тд)

## Что такое Nashorn?

первое -- он не используется в современных версиях
а так -- движок JavaScript, разработан на Java, разрабатывала его Oracle

Цель: встраивать код JS в Java

Nashron компилирует код и передает байт-код в JVM непосредтсвенно в памяти

Nashorn умеет компилировать код JavaScript и генерировать классы Java, которые загружаются специальным загрузчиком. 

Так же возможен вызов кода Java прямо из JavaScript.

## Что такое jjs?

jjs это утилита командной строки, которая позволяет исполнять программы на языке JavaScript прямо в консоли.

## Какой класс появился в Java 8 для кодирования/декодирования данных?

можно встретить в читалках (кодирование / декодирования в формат эл. книг)

`Base64` - потокобезопасный класс, который реализует кодировщик и декодировщик данных, используя схему кодирования base64 согласно RFC 4648 и RFC 2045.

- getEncoder()/getDecoder() - возвращает кодировщик/декодировщик base64, соответствующий стандарту RFC 4648;

- getUrlEncoder()/getUrlDecoder() - возвращает URL-safe кодировщик/декодировщик base64, соответствующий стандарту RFC 4648;

- getMimeEncoder()/getMimeDecoder() - возвращает MIME кодировщик/декодировщик, соответствующий стандарту RFC 2045.


```java
String b64 = Base64.getEncoder().encodeToString("input".getBytes("utf-8")); // aW5wdXQ==
```

```java
new String(Base64.getDecoder().decode("aW5wdXQ=="), "utf-8"); // input
```

## Какие дополнительные методы для работы с ассоциативными массивами (maps) появилисьв Java 8?
то есть у на сесть Map<K,V> map и его метолы map.\<method>:
(далее все Параметры схематичны, не точные)

- `putIfAbsent(Key, Value)` -- добавляет пару ключ-значение, только если ключа не было 

- `forEach((k,v) -> ...)` -- принимает функцию (тут я написал как пример), которая производит операции над каждым элементом

- `compute(Key, Function(Key, Value))` -- создает или обновляет текущее значение на полученное в результате вычисления (возможно использовать ключ и текущее значение)

- `computeIfPresent()` -- делает тоже самое, но только если ключ существует

- `computeIfAbsent()` -- если ключ отсуствует, то тогда создает его со значением, которые вычисляется 

- `getOrDefault(key, defaultValue)` -- в случае остуствия ключа, возвращает переданное значение по-умолчанию

- `merge(key, value, function(value, newValue))` -- объеденяет передаваемое и текущее значение. если под заданным ключем значение отсуствует, то записывает туда передаваемое значение
